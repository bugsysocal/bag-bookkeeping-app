## Standing design rules (from reviews)
- Double-entry rigor under the hood, owner-friendly surface; nothing posted is ever deleted (void/reverse only).
- Compliance-critical settings: **advisor-gated edits, owner-facing visibility** (banners derived from ledger data) — never owner edit power.
- Tax rates/thresholds are **data, not code**; reminders follow the fiscal year, not the calendar year.

## For a new developer joining this project

You are inheriting a **spec-first** codebase: every feature was written as a reviewed markdown spec in `specs/` **before** any code. When in doubt about intended behaviour, the spec is the source of truth, and [specs/00-key-decisions-digest.md](specs/00-key-decisions-digest.md) is the one-page map to all of it. This file (PROGRESS.md) is the authoritative *status* doc — what is built, what is not, what to do next.

### Build & run from a clean clone
Prerequisites: **Rust** (stable, MSVC toolchain), **Node.js** (only needed if you regenerate Tauri assets; the committed `ui/` is plain HTML/JS and needs no build step). No database setup — SQLite is embedded (`rusqlite` `bundled` feature) and the schema is created on first run by the migration runner in `crates/ledger-core/src/db.rs`.

```
cargo test --workspace          # run the whole suite (currently 43 tests)
cargo build -p ledgerone-desktop   # build the desktop app
cargo run   -p ledgerone-desktop   # launch it (creates its DB under %APPDATA%\ng.boltonadvisory.ledgerone)
```
The running app's data lives at `%APPDATA%\ng.boltonadvisory.ledgerone\ledgerone.db`. To reset to a first-run state, rename/move that file (do not delete blindly — it is real books once used).

### Windows / ARM64 toolchain gotchas (this dev machine is Windows on ARM64)
These bit the previous developer; they will bite you too on a similar machine:
- **`VC.Tools.ARM64` MSVC component**: the VS 2022 "VCTools" workload installs x64-host tools by default. On an ARM64 host you must add the ARM64 component separately or linking fails with "link.exe not found". Install via winget with `--force --override "... --add Microsoft.VisualStudio.Component.VC.Tools.ARM64"`; the VS installer's own `setup.exe --quiet` is blocked from a non-elevated shell (exit 5007) — go through winget.
- **rustup `settings.toml` conflict**: if a stale `~/.rustup/settings.toml` points at a toolchain that no longer exists, `cargo` errors before it does anything. Fix by reinstalling the stable toolchain (`rustup toolchain install stable-aarch64-pc-windows-msvc`) or clearing the stale default.
- **`cargo` on PATH**: lives at `%USERPROFILE%\.cargo\bin`. Pre-existing shells may not have it until PATH is refreshed; call it by full path if `cargo` is "not recognized".
- **Windows Defender + `target/`**: Defender scans freshly-linked binaries and intermittently holds a lock, causing `LNK1104`/`LNK1201`. **Add the project `target/` folder to Defender exclusions** (needs admin) — this is the durable fix. Stop-gap without admin: `Remove-Item target\debug\deps\*.exe, target\debug\deps\*.pdb` then rebuild.
- **`chrono` is banned from `ledger-core`**: its `num-traits`/`autocfg` build script fails in this environment. Timestamps are hand-rolled std-only civil-date math in `ids.rs` — do not reintroduce chrono.

### Architectural boundary — do not cross it
This is the single most important rule in the codebase:

> **All posting/business logic lives in `crates/ledger-core`. The Tauri shell (`src-tauri/src/lib.rs`) contains only thin command wrappers that (de)serialize DTOs and call `ledger-core`. Never write ledger logic, SQL that posts, or accounting rules in the shell.**

- Every financial event flows through one posting surface (`engine.rs` functions → `posting::post_entry_in`) inside one DB transaction; the balance guarantee is enforced by SQLite triggers (Spec 01 §4), not by app code. See [specs/00-key-decisions-digest.md](specs/00-key-decisions-digest.md) "five invariants."
- Errors returned to the UI use the `CmdError` envelope: an owner-language message + a next step, with any technical detail carried separately for Advisor Mode. Every new command must keep this — no raw engine strings reach the owner.
- The frontend (`ui/index.html`) is a single dependency-free HTML/JS file talking to commands via `window.__TAURI__.core.invoke`. Its only capabilities are core IPC + the opener plugin (wa.me/mailto). No shell-exec, no fs, no arbitrary network.
